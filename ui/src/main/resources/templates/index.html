Song 1 — Plug & Play

Auto-registration with Consul

  • Add stork-service-registration-consul dependency
  • Configure registrar in application.properties
  • Service registers itself on startup — zero code

# application.properties
quarkus.stork.band-service.service-registrar.type=consul
quarkus.stork.band-service.service-registrar.consul-host=localhost
quarkus.stork.band-service.service-registrar.consul-port=8500

Song 2 — Service Discovery

Consul discovery with instance caching

  • Stork discovers instances from Consul
  • Caches the list — no network call per request
  • Cache refreshes every refresh-period (default: 5 min)

quarkus.stork.band-service.service-discovery.type=consul
quarkus.stork.band-service.service-discovery.consul-host=localhost
quarkus.stork.band-service.load-balancer.type=round-robin

Song 2 — Live Demo

Caching: On the first call, Stork fetches instances from Consul and caches them. Subsequent calls use the cache.

Try it: Start a new guitarist while this is running. It won't appear until the cache refreshes.

Song 3 — Battle of Strategies

Live Load Balancing comparison

  • round-robin — sequential distribution
  • random — probabilistic
  • least-requests — min in-flight count
  • power-of-two-choices — min of 2 random picks
  • least-response-time — time-weighted scoring
  • sticky — session affinity with failure backoff
