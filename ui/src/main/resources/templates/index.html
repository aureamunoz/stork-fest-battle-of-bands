Song 2 — Service Discovery

Consul discovery with instance caching

Stork discovers instances from Consul and caches them to avoid calling the registry on every request. The cache refreshes periodically (refresh-period).

How caching works:

On the first call, Stork fetches the instance list from Consul and caches it. Subsequent calls use the cache — no extra network round-trip to Consul. The cache is refreshed every refresh-period (default: 5 min).

Try it: Start a new guitarist while this is running. It won't appear immediately — only after the cache refreshes.
